## 2. АРХИТЕКТУРА СИСТЕМЫ

### 2.1 Проверка на плагиат

#### A. Локальная база данных (основной метод)
```
1. Пользователь загружает документ
2. Извлекаем текст (PDF/DOCX → plain text)
3. Разбиваем на фрагменты (shingles) по 5-10 слов
4. Создаем fingerprint (хэш) каждого фрагмента
5. Ищем совпадения в БД
6. Сохраняем новый документ в БД для будущих проверок
```

**Преимущества:**
- Бесплатно
- Быстро
- Точность растет со временем
- Находит плагиат между студентами

#### B. Поиск через интернет (дополнительный метод)
```
1. Берем уникальные фрагменты текста (3-5 предложений)
2. Делаем поисковые запросы через:
   - Google Custom Search API (100 запросов/день бесплатно)
   - Парсинг Яндекса (через прокси, ротация)
3. Анализируем найденные страницы
4. Сравниваем с исходным текстом
```

**Ограничения:**
- Лимиты API
- Может блокироваться при парсинге
- Медленнее локальной проверки

#### C. Fingerprinting алгоритм
```javascript
// Пример алгоритма
function createFingerprint(text) {
  // 1. Нормализация
  const normalized = text.toLowerCase()
    .replace(/[^\wа-яё\s]/gi, '')
    .replace(/\s+/g, ' ');
  
  // 2. Создание shingles (скользящее окно)
  const shingles = [];
  const words = normalized.split(' ');
  const windowSize = 7;
  
  for (let i = 0; i <= words.length - windowSize; i++) {
    const shingle = words.slice(i, i + windowSize).join(' ');
    shingles.push(hash(shingle));
  }
  
  // 3. MinHash для быстрого сравнения
  return createMinHash(shingles);
}
```

### 2.2 Проверка на AI-генерацию

#### Методы детекции:

**A. Статистический анализ**
```
- Средняя длина предложений (AI часто делает равномерные)
- Разнообразие словаря (vocabulary richness)
- Частота редких слов
- Паттерны пунктуации
```

**B. Перплексия (Perplexity)**
```
- Мера "предсказуемости" текста
- AI тексты обычно имеют низкую перплексию
- Используем простую языковую модель для оценки
```

**C. Характерные паттерны AI**
```
- Частые фразы: "в заключение", "таким образом", "следует отметить"
- Идеальная структура (введение → основная часть → заключение)
- Отсутствие опечаток и грамматических ошибок
- Слишком формальный стиль
```

**D. Эвристики**
```javascript
function detectAI(text) {
  let aiScore = 0;
  
  // 1. Проверка на типичные AI фразы
  const aiPhrases = [
    'в заключение можно сказать',
    'таким образом',
    'следует отметить',
    'важно подчеркнуть',
    'в современном мире'
  ];
  
  aiPhrases.forEach(phrase => {
    if (text.includes(phrase)) aiScore += 10;
  });
  
  // 2. Анализ разнообразия
  const words = text.split(/\s+/);
  const uniqueWords = new Set(words);
  const diversity = uniqueWords.size / words.length;
  
  if (diversity < 0.4) aiScore += 20; // Низкое разнообразие
  
  // 3. Проверка структуры
  const hasIntro = /введение|вступление/i.test(text);
  const hasConclusion = /заключение|вывод/i.test(text);
  if (hasIntro && hasConclusion) aiScore += 15;
  
  // 4. Отсутствие ошибок (подозрительно для студента)
  const typos = detectTypos(text);
  if (typos < 2 && words.length > 1000) aiScore += 15;
  
  return Math.min(aiScore, 100);
}
```

## 4. СТРУКТУРА БАЗЫ ДАННЫХ

```sql
-- Таблица документов
CREATE TABLE documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(255),
  filename VARCHAR(255),
  file_size INTEGER,
  file_type VARCHAR(10),
  text_content TEXT,
  word_count INTEGER,
  created_at TIMESTAMP DEFAULT NOW(),
  INDEX idx_created (created_at)
);

-- Таблица фрагментов (shingles)
CREATE TABLE shingles (
  id BIGSERIAL PRIMARY KEY,
  document_id UUID REFERENCES documents(id) ON DELETE CASCADE,
  shingle_hash BIGINT,
  position INTEGER,
  INDEX idx_hash (shingle_hash)
);

-- Таблица результатов проверки
CREATE TABLE check_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID REFERENCES documents(id),
  plagiarism_percent DECIMAL(5,2),
  ai_detection_percent DECIMAL(5,2),
  sources JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Таблица совпадений
CREATE TABLE matches (
  id BIGSERIAL PRIMARY KEY,
  source_doc_id UUID REFERENCES documents(id),
  target_doc_id UUID REFERENCES documents(id),
  match_percent DECIMAL(5,2),
  matched_fragments JSONB
);
```

### Точность системы:

**Плагиат:**
- Локальная БД: 70-80% (растет со временем)
- Интернет-поиск: 50-60% (лимиты API)
- **Общая: 60-75%** vs 90%+ у коммерческих

**AI детекция:**
- Статистические методы: 60-70%
- Эвристики: 65-75%
- **Общая: 60-75%** (это сложная задача)

### Ограничения:
- Не найдет плагиат из закрытых источников
- Может давать ложные срабатывания
- AI детекция не 100% точная
- Лимиты бесплатного хостинга

### Преимущества:
- Полностью бесплатно для пользователей (базовый тариф)
- База растет → точность улучшается
- Открытый код → доверие студентов
